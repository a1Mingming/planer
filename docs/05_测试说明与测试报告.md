# 测试说明与测试报告

## 1. 测试目标

验证核心 API 接口正确响应，前端三个视图可正常访问、交互符合需求，移动端布局正确切换，主要边界条件（缺字段、非法参数、重复标签）按预期报错。补充自动化测试，覆盖 Service 层业务逻辑、Repository 层数据转换、前端交互关键路径。

## 2. 测试范围

### 已覆盖

- 后端所有 REST 接口（curl 手工验证）
- 前端三级视图基本渲染
- 计划 CRUD 完整流程
- 标签查询与创建
- 缺少必填字段的错误响应
- 移动端布局切换（Chrome DevTools 模拟）
- 代理转发正确性
- **PlanService / TagService 单元测试**（vitest，mock repository）
- **zod validators 单元测试**（CreatePlanSchema / UpdatePlanSchema / GetPlansSchema / CreateTagSchema）
- **SqlitePlanRepository / SqliteTagRepository 集成测试**（SQLite `:memory:`）
- **前端 E2E 测试**（Playwright，API 通过 `page.route` mock）

### 未覆盖

- 并发写入压力测试
- 跨浏览器兼容性（E2E 仅验证 Chromium）
- 生产构建产物验证

## 8. 问题与修复记录

| 问题 | 发现方式 | 原因 | 修复方式 | 复验结果 |
|---|---|---|---|---|
| 后端启动报"目录不存在" | 首次运行 tsx | `getDb()` 在 `mkdirSync` 前被调用，data/ 目录不存在 | 将 `mkdirSync` 移入 `connection.ts` 的 `getDb()` 中，首次打开前自动创建 | ✅ 已修复 |
| 前端出现双层导航栏 | 浏览器访问 | Umi 4 自动将 `src/layouts/index.tsx` 注册为全局布局，路由配置中又手动声明了 component，导致两次渲染 | 路由配置中去掉手动的 layout 包装，交由 Umi 约定机制处理 | ✅ 已修复 |
| 前端代理 /api 返回 404 | curl 访问 :8000 | nvm-windows 旧进程残留，占用 `::1`（IPv6）3001 端口；Umi 代理用 `localhost` 解析到 IPv6，命中无路由的旧进程 | 代理目标从 `localhost` 改为 `127.0.0.1`，强制走 IPv4 | ✅ 已修复 |
| pnpm 安装 better-sqlite3 失败 | 安装时报 node-gyp 错误 | bash wrapper 拦截 pnpm install，内部 node-gyp 使用 Node 16 编译原生模块 | bash wrapper 已移除；package.json 添加 `onlyBuiltDependencies` | ✅ 已修复 |
| Umi 配置 antd/layout 报错 | umi setup 输出 | plain `umi` 不支持 `antd`、`layout` 配置项，这是 `@umijs/max` 特有配置 | 从 .umirc.ts 移除这两个配置项 | ✅ 已修复 |
| PlanForm 弹窗整页出现滚动条 | 手动测试 | `.modalWrap` 在 `.ant-modal-wrap`（fixed 全屏容器）上加 `overflow-y: auto`；AntD6 将 `.ant-modal-content` 更名为 `.ant-modal-container` 导致 CSS 未命中 | 删除 `.modalWrap`，选择器适配 `ant-modal-container`，body `overflow-y: auto` + flex column 实现内部滚动 | ✅ 已修复 |
| PlanForm footer 按钮与表单重叠 | 手动测试 | footer 缺少 `background` 色值，滚动时透明导致视觉重叠 | 添加 `background: var(--bg-surface)` + `align-items: center` + `position: sticky` | ✅ 已修复 |
