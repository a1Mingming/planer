# 部署说明

## 1. 环境要求

| 依赖 | 版本 / 要求 |
|---|---|
| Node.js | 20.x（必须，better-sqlite3 原生模块编译依赖） |
| pnpm | 任意版本（推荐 8+） |
| Windows C++ 构建工具 | 编译 better-sqlite3 所需（已安装则跳过） |
| 操作系统 | Windows / macOS / Linux（本项目在 Windows 11 开发验证） |

> **nvm-windows 用户**：安装依赖前执行 `nvmuse 20` 切换到 Node 20。

## 2. 配置说明

| 配置项 | 默认值 | 说明 |
|---|---|---|
| 后端端口 | 3001 | `src/interface/index.ts` 中 `process.env.PORT ?? 3001` |
| 前端端口 | 8000 | Umi 默认端口 |
| 数据库路径 | `server/data/plans.db` | `process.env.DB_PATH` 可覆盖；目录首次启动自动创建 |
| API 代理 | `/api → http://127.0.0.1:3001` | `client/.umirc.ts` proxy 配置 |

无需额外 `.env` 文件，开箱即用。

## 3. 本地启动步骤

```bash
# ── 后端 ──────────────────────────────────
# 1. 进入后端目录
cd server

# 2. 安装依赖（Node 20 环境下）
nvmuse 20          # nvm-windows 用户执行，其他环境确保 node -v 为 20.x 即可
pnpm install

# 3. 启动开发服务器（tsx watch 热重载）
pnpm dev
# 输出：[server] running on http://localhost:3001
# 首次启动自动创建 data/plans.db 并执行 migrate + seed

# ── 前端 ──────────────────────────────────
# 4. 新开终端，进入前端目录
cd client

# 5. 安装依赖
nvmuse 20
pnpm install

# 6. 启动开发服务器
pnpm dev
# 输出：App listening at http://localhost:8000

# ── 访问 ──────────────────────────────────
# 7. 浏览器打开
http://localhost:8000
```

## 4. Docker 部署（一键启动）

```bash
# 从项目根目录（需已安装 Docker Desktop）
docker compose up --build
```

首次构建约需 3–5 分钟（下载基础镜像、编译 TypeScript、安装 npm 依赖、构建前端）。完成后访问 `http://localhost`。

### 架构

```
Browser :80
  └── frontend 容器（Nginx）
        ├── /api/*  → 转发到 backend:3001（Express 容器）
        └── /*      → 静态文件（Umi build 产物）
```

后端容器不对外暴露端口，仅通过 Docker 内部网络供 Nginx 访问。SQLite 数据库通过 named volume `sqlite_data` 持久化。

### 常用命令

```bash
# 后台运行（已构建过）
docker compose up -d

# 查看日志
docker compose logs -f

# 停止容器（保留数据库）
docker compose down

# 停止并删除数据库 volume（完全重置）
docker compose down -v

# 重新构建（修改代码后）
docker compose up --build
```

### 端口说明

| 容器 | 内部端口 | 对外暴露 |
|------|---------|---------|
| frontend（Nginx） | 80 | :80 |
| backend（Express） | 3001 | 不暴露（仅内部） |

## 5. 测试执行方式

```bash
# 后端类型检查
cd server
pnpm lint      # tsc --noEmit，无输出即通过

# 后端自动化测试（先停止 dev server，再安装依赖）
cd server
pnpm install   # 首次需安装 vitest
pnpm test      # 单元 + 集成测试
pnpm test:coverage  # 覆盖率报告

# 接口冒烟测试（后端启动后执行）
curl "http://127.0.0.1:3001/api/tags"
# 预期：{"success":true,"data":[5 个预置标签]}

curl "http://127.0.0.1:3001/api/plans?view=month&date=2026-06"
# 预期：{"success":true,"data":[]}

# 通过前端代理测试
curl "http://127.0.0.1:8000/api/tags"
# 预期：同上（HTTP 200）

# 前端 E2E 测试（先启动前后端 dev server，再执行）
cd client
pnpm install                      # 首次需安装 @playwright/test
npx playwright install chromium   # 首次下载 Chromium 浏览器
pnpm test:e2e                     # 有头模式（可见浏览器窗口）
pnpm test:e2e:ui                  # Playwright UI 交互模式（推荐调试）
```

## 6. 演示数据

启动时 `seed.ts` 自动写入 5 个预置标签：**工作、学习、健身、生活、其他**（`INSERT OR IGNORE`，重复启动安全）。

无需手动初始化数据，启动即可使用。若需要测试数据，可通过前端表单或 curl 创建：

```bash
curl -X POST http://127.0.0.1:3001/api/plans \
  -H "Content-Type: application/json" \
  -d '{"title":"示例计划","date":"2026-06-29","start_time":"09:00","end_time":"10:00","tags":["工作"],"done":false}'
```

## 7. 常见问题

| 问题 | 解决方式 |
|---|---|
| `better-sqlite3` 安装报 node-gyp 错误 | 确认 Node 版本为 20；Windows 需安装 C++ 构建工具：管理员 PowerShell 执行 `npm install --global windows-build-tools` |
| 后端启动报 `EADDRINUSE: 3001` | 端口被占用，执行 `netstat -ano \| grep ":3001 "` 找到 PID，`taskkill /PID <pid> /F` 终止 |
| 前端 /api 接口返回 404 | 检查后端是否启动；若存在旧 Node 进程残留，将 `.umirc.ts` 代理目标改为 `http://127.0.0.1:3001` |
| pnpm 命令未找到 | 执行 `npm install -g pnpm --registry https://registry.npmmirror.com` 全局安装 |
| Umi setup 报 "Node 20 is required" | 当前 shell 的 node 版本不对，执行 `nvmuse 20` 后重试；或直接用 `node ./node_modules/umi/bin/umi.js setup` |
| 前端构建后访问空白 | 确认 `.umirc.ts` routes 配置正确，无语法错误；执行 `pnpm dev` 查看控制台报错 |
| pnpm install 报 EPERM（better-sqlite3） | `.node` 文件被 dev server 进程占用，先停止后端 dev server 再安装 |
